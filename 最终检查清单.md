## 🚀 GitHub Pages 部署步骤

### 步骤 1：创建仓库

1. 访问 [github.com](https://github.com) 并登录
2. 点击 **+** → **New repository**
3. 仓库名：`rongyun-diy`（或您喜欢的名称）
4. 选择 **Public**
5. 点击 **Create repository**

### 步骤 2：上传文件

1. 在仓库页面点击 **uploading an existing file**
2. 打开 `c:\Users\15759\Desktop\DIY` 文件夹
3. 拖入以下内容：
   - `index-standalone.html`
   - `admin.html`
   - `src/` 文件夹（完整拖入）
   - `图画/` 文件夹（完整拖入）
   - `styles.css`
   - `README.md`（可选）
4. 填写提交信息：`Initial commit`
5. 点击 **Commit changes**

### 步骤 3：开启 Pages

1. 在仓库页面点击 **Settings**
2. 滚动找到 **Pages**
3. 在 **Branch** 下拉选择 `main`（或 `master`）
4. 点击 **Save**
5. 等待 2-5 分钟

### 步骤 4：获取链接

部署完成后，会显示访问链接：
- 游客访问：`https://您的用户名.github.io/rongyun-diy/index-standalone.html`
- 管理端：`https://您的用户名.github.io/rongyun-diy/admin.html`

## 🔧 常见问题

### Q: 图片显示不出来？
A: 确保 `图画/` 文件夹完整上传了，并且路径正确

### Q: 订单无法提交？
A: 检查 Supabase URL 和密钥，确保网络通畅

### Q: 管理端看不到订单？
A: 检查 Supabase 配置是否一致，等待几秒钟自动同步
