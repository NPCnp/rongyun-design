# 更新日志

## v1.2.0 (2026-05-30)

### 🔧 修复
- **修复 admin.html Supabase 同步功能**
  - 修复了从 Supabase 同步订单时没有保存订单 ID 的问题
  - 现在标记完成/待制作时会自动同步到 Supabase
  - 现在删除订单时会自动同步到 Supabase
  - 批量操作也支持同步到 Supabase

### 📁 项目整理
- **文件夹结构优化**
  - 将所有文档文件整理到 `docs/` 文件夹
  - 将所有构建脚本整理到 `build/` 文件夹
  - 将旧版本文件整理到 `backup/` 文件夹
  - 保留根目录的核心文件便于使用

### 📝 配置更新
- 修复了 `src/config.js` 中的 Supabase URL 配置
- 修复了 `admin.html` 中的 Supabase 配置
- 更新了 `src/order-manager.js` 中的 Supabase 初始化代码

## v1.1.0 (2026-05-29)

### ✨ 新功能
- **集成 Supabase 云存储**
  - 支持云端保存和同步订单数据
  - 提供本地存储和云端存储双重保障
  - 多设备自动同步订单信息
  - 创建了 `supabase-setup.sql` 数据库初始化脚本

### 🔧 优化
- **订单状态修复**
  - 统一了状态值为 "待制作" 和 "已制作"
  - 修复了状态筛选逻辑
  - 添加了状态统一函数 `normalizeOrderStatus()`

### 📁 文件重构
- 将 `index-standalone.html` 拆分为多个文件：
  - `src/config.js` - 配置文件
  - `src/data.js` - 数据文件
  - `src/canvas-manager.js` - 画布管理
  - `src/order-manager.js` - 订单管理
  - `src/app.js` - 主应用逻辑
  - `src/styles.css` - 样式文件
